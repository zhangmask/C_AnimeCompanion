# CompanionChat 记忆用户视角归属修复设计

> 日期：2026-05-13
> 背景：阶段三记忆系统已生效，但真机验证发现“用户记忆被模型误当成自身信息”的视角错误。

## 1. 问题描述

当前长期记忆和动态检索记忆都会被拼接进 system prompt，但记忆段只提供了标题和条目列表，没有明确约束这些内容的归属主体。

例如用户手动写入：

```text
小王是我的哥哥
```

模型可能将其中的“我”理解为助手自身，而不是理解为“用户”。

这会导致：

- 记忆已经被检索和注入，但回答视角错误
- 关系类记忆最容易出现主体混淆
- 后续更多“我/我的”表达都可能复现同类问题

## 2. 根因判断

根因不在数据库、FTS、Room 或生命周期管理，而在 prompt 表达层：

- `MemoryPromptBuilder` 当前只输出“长期记忆中的关键信息”之类的标题
- `PromptAssembler` 只是机械拼接各段内容
- 没有显式规则告诉模型：
  - 这些记忆都属于用户
  - 记忆中的“我”默认指用户
  - 不应将这些信息解释成助手自身设定

因此这属于“记忆已生效，但提示词约束不足”的问题。

## 3. 修复目标

本次修复要达到以下结果：

- 明确所有记忆默认属于用户，而不是助手
- 明确记忆中的“我/我的”默认指用户
- 不改数据库结构，不改检索策略，不改记忆内容本身
- 通过最小范围修改降低对现有阶段三链路的影响

## 4. 方案选择

### 方案 A：只改记忆条目内容

示例：

```text
- [关系] 用户说：小王是我的哥哥
```

优点：

- 修改小

缺点：

- 约束力度仍然不够强
- 只是“像在提示”，不是明确规则

### 方案 B：只在 system prompt 增加全局解释规则

示例：

```text
以下记忆都属于用户本人，不属于助手。除非用户明确要求角色扮演，其中“我”一律指用户。
```

优点：

- 指令性强
- 对模型行为约束更直接

缺点：

- 记忆段本身仍然缺少归属提示

### 方案 C：全局规则 + 记忆段标签一起补

结论：采用本方案。

做法：

- 在 `PromptAssembler` 中，当存在任一记忆段时，插入统一的“记忆解释规则”
- 在 `MemoryPromptBuilder` 中，为记忆段增加“以下内容均为用户信息”的局部说明

原因：

- 全局规则负责强约束
- 局部说明负责增强可读性和稳定性
- 保持记忆原文不变，避免过度改写用户写入内容

## 5. 详细设计

### 5.1 PromptAssembler 增加记忆解释规则

当 `persistentMemoryPrompt` 或 `memoryPrompt` 任一非空时，插入一段统一规则：

```text
记忆解释规则：
- 以下记忆都描述用户本人的信息、关系、偏好或经历，不是助手自己的信息。
- 除非用户明确要求角色扮演、改写文案或切换叙述视角，记忆中的“我”“我的”默认都指用户。
- 回答涉及这些记忆时，应使用“你”或“用户”的视角理解和表达。
```

插入顺序：

1. 基础 system prompt
2. 用户偏好
3. 记忆解释规则
4. 常驻长期记忆段
5. 动态检索记忆段
6. 历史摘要
7. 最近对话片段

### 5.2 MemoryPromptBuilder 增加局部说明

在每个记忆段标题下追加一行局部说明：

```text
以下内容均为用户本人的记忆，不代表助手自身。
```

示例：

```text
长期记忆中的关键信息：
以下内容均为用户本人的记忆，不代表助手自身。
- [关系] 小王是我的哥哥
```

这样即使模型更关注局部段落，也能再次看到主体约束。

## 6. 测试策略

本次只补 prompt 层测试，不引入新的集成测试。

### 6.1 MemoryPromptBuilderTest

增加或调整断言：

- 长期记忆段包含“以下内容均为用户本人的记忆”
- 动态记忆段也包含同样的局部说明
- 原有分类映射和空字符串行为不变

### 6.2 PromptAssemblerTest

增加或调整断言：

- 有记忆段时会插入“记忆解释规则”
- 无记忆段时不会插入该规则
- 规则顺序位于用户偏好之后、记忆段之前

## 7. 风险与控制

### 风险 1：prompt 变长

影响较小，因为新增的是固定短说明。

### 风险 2：规则重复

全局规则和局部说明会有一定重复，但这是有意的冗余，用来提升模型稳定理解。

### 风险 3：仍可能个别回答跑偏

本次修复先解决“默认主体归属不清”的主问题；如果后续仍有极个别模型输出偏差，再考虑对关系类记忆做专门模板化表达。

## 8. 结论

本次采用“全局记忆解释规则 + 记忆段局部说明”的最小修复方案。

它不改变存储层和检索层，只增强 prompt 中对主体归属的约束，目标是让模型稳定把“我的哥哥”“我的偏好”“我的经历”理解为用户信息，而不是助手自身信息。
