# Anime Companion 隐私与安全 (MNN 版本)

> **隐私是物理保证，不是政策承诺。**

## 核心隐私承诺

| 承诺 | 实现方式 |
|------|----------|
| **对话不离开设备** | 所有推理在本地 MNN 完成，无云端 API 调用 |
| **数据不被收集** | 无用户账号系统，无数据上传，无行为分析 |
| **模型本地加载** | Qwen3.5 2B 模型文件存储在本地，不需要联网验证 |
| **记忆本地存储** | 所有记忆数据存储在本地 SQLite，不上传任何服务器 |
| **网络权限可选** | 网络权限仅用于可选的 HTTP 图片生成，用户可完全关闭 |

## 隐私架构

```
用户数据
├── 对话数据 ──→ 本地 Room SQLite ──→ 仅在设备上
├── 记忆数据 ──→ 本地 SQLite + FTS4 ──→ 仅在设备上
├── 偏好数据 ──→ 本地 SharedPreferences ──→ 仅在设备上
├── 语音数据 ──→ 本地 MNN/ONNX 推理 ──→ 处理后立即丢弃
└── 模型文件 ──→ 本地存储 ──→ 不联网验证
```

## 技术保证

### 1. 端侧推理

- 主对话模型 (Qwen3.5 2B) 在本地 MNN 推理
- 语音识别 (SenseVoice) 在本地 ONNX 推理
- 语音合成 (MOSS TTS) 在本地 ONNX 推理
- 文本嵌入 (Qwen3.5 2B Embedding) 在本地 MNN 推理
- 所有推理都在设备上完成，不调用任何云端 API

### 2. 本地存储

- 对话历史：Room SQLite 数据库
- 记忆数据：SQLite + FTS4 全文索引
- 用户偏好：SharedPreferences
- 角色卡：Room SQLite 数据库
- 所有数据都存储在应用私有目录，其他应用无法访问

### 3. 网络隔离

- 网络权限仅用于可选的 HTTP 图片生成功能
- 用户可以在设置中完全关闭网络功能
- 关闭网络后，所有核心功能（对话、记忆、语音）完全正常
- 没有用户账号系统，不需要注册

### 4. 数据加密

- Android 全盘加密 (FBE) 默认开启
- 应用私有目录受系统级保护
- 用户可选择启用应用级数据库加密 (SQLCipher)

## MNN 版本的隐私优势

| 对比 | MNN 版本 | 云端 AI 服务 |
|------|----------|-------------|
| 数据存储 | 100% 本地 | 云端服务器 |
| 推理位置 | 设备上 | 云端 |
| 网络依赖 | 可选 | 必须 |
| 数据收集 | 无 | 有 |
| 隐私保证 | **物理保证** | 政策承诺 |

| CCPA (加州) | **合规**：不收集个人信息 |
| 中国个保法 | **合规**：数据本地存储，不跨境传输 |

## 用户权利

| 权利 | 实现方式 |
|------|----------|
| 查看数据 | App 内可查看所有对话、记忆、偏好 |
| 删除数据 | App 内可删除单条/全部对话、记忆、偏好 |
| 导出数据 | 支持导出对话历史和记忆数据 |
| 关闭功能 | 可关闭记忆提取、偏好学习、语音等 |
| 网络控制 | 可完全禁止 App 的网络访问 |

## 隐私承诺

**我们的隐私保护是架构级的，不是政策级的。**

即使我们想要收集用户数据，从技术架构上也做不到：

1. 没有用户账号系统，无法关联数据
2. 没有数据上传接口，无法传输数据
3. 没有云端服务器，无法存储数据
4. 所有推理在本地完成，数据不出设备

**这是"隐私是物理保证"的含义。**
